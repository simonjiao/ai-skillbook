# Contributing

This repository is a practical notebook. Prefer small, specific entries over broad essays.

## Entry Guidelines

- State the context where the technique or tool applies.
- Include setup steps only when they are needed to reproduce the result.
- Record failure modes and tradeoffs.
- Link to source documents, official docs, or local examples when possible.
- Keep generated outputs separate from reusable templates.

## Review Checklist

- The entry has front matter.
- Claims are specific enough to verify.
- Public notes do not include secrets, private endpoints, tokens, or personal data.
- The recommendation distinguishes confirmed evidence from inference.
